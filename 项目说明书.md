# Tinc VPN 客户端配置工具 - 项目说明书

## 1. 项目概述

### 1.1 项目简介
本项目是一个基于 Qt 5.15.2 框架开发的 Tinc VPN 客户端配置工具，提供图形化用户界面（GUI）用于用户登录、配置下载、密钥生成和服务器注册等功能。项目支持 Windows 和 Linux 平台，实现了 Tinc VPN 的自动化配置流程。

### 1.2 主要功能
- 用户登录与身份验证
- 自动下载 Tinc 配置文件
- 自动生成 RSA 密钥对
- 公钥上传与服务器注册
- 配置进度实时显示
- 详细的日志记录
- 后台守护进程管理

### 1.3 技术栈
- **开发框架**: Qt 5.15.2 (Core, Network, Widgets, Gui)
- **编程语言**: C++
- **编译器**: MinGW 8.1.0 (Windows) / GCC (Linux)
- **VPN软件**: Tinc 1.1
- **网络协议**: HTTP/HTTPS

## 3. 核心模块详解

### 3.1 登录模块 (LoginDialog)

#### 3.1.1 功能说明
提供用户登录界面，验证用户身份并启动配置流程。

#### 3.1.2 主要文件
- **logindialog.cpp/h**: 登录界面实现
- **download.cpp/h**: 配置下载与处理逻辑

#### 3.1.3 工作流程
1. 用户输入用户名和密码
2. 向服务器发送登录请求
3. 验证成功后获取认证令牌（token）
4. 调用配置下载模块
5. 启动配置向导界面

### 3.2 配置下载模块 (download)

#### 3.2.1 功能说明
负责从服务器下载 Tinc 配置文件，并启动密钥生成和上传流程。

#### 3.2.2 主要功能
- 下载配置文件压缩包
- 解压配置文件到指定目录
- 启动 conf_package.exe 生成密钥
- 监控配置进度
- 显示配置日志

#### 3.2.3 工作流程
1. 接收登录成功后的参数（sid, token, serverIp）
2. 构建下载 API 请求
3. 下载配置文件压缩包
4. 解压到 Tinc 安装目录
5. 启动 conf_package.exe 生成密钥
6. 等待密钥生成和上传完成
7. 更新配置进度

#### 3.2.4 进度说明
- **0%**: 初始化完成，开始下载
- **10%**: 配置文件下载完成
- **20%**: 开始生成密钥
- **50%**: 密钥生成完成
- **80%**: 开始上传公钥
- **100%**: 配置完成

### 3.3 配置打包模块 (conf_package)

#### 3.3.1 功能说明
负责生成 RSA 密钥对，并将公钥上传到服务器完成节点注册。

#### 3.3.2 主要功能
- 读取 private.txt 配置文件
- 调用 tincd.exe 生成 RSA 密钥对
- 读取生成的公钥文件
- 构造 JSON 格式的上传请求
- 上传公钥到服务器
- 记录详细日志

#### 3.3.3 工作流程
1. 从命令行参数接收配置信息
2. 从 private.txt 读取服务器配置
3. 调用 tincd.exe 生成密钥对
4. 处理 tincd 的交互式提示
5. 读取生成的公钥文件
6. 构造 HTTP POST 请求
7. 上传公钥到服务器
8. 处理服务器响应

#### 3.3.4 日志功能
- 日志文件位置: `conf_package.exe` 所在目录
- 日志文件名: `conf_log.txt`
- 日志内容:
  - 接收到的参数
  - 密钥生成过程
  - 公钥上传过程
  - 服务器响应
  - 错误信息

### 3.4 配置向导模块 (confg)

#### 3.4.1 功能说明
提供图形化的配置向导界面，显示配置进度和日志信息。

#### 3.4.2 主要功能
- 显示配置向导标题
- 显示配置进度条
- 显示配置日志信息
- 提供开始配置按钮
- 显示配置完成状态

#### 3.4.3 界面元素
- 标题: "Tinc内网配置向导"
- 进度条: 显示配置进度（0-100%）
- 日志显示: 实时显示配置过程
- 开始按钮: 启动配置流程

### 3.5 后台守护进程 (Daemons)

#### 3.5.1 功能说明
作为系统服务运行，负责监控和管理 Tinc 进程。

#### 3.5.2 主要功能
- 启动 Tinc VPN 服务
- 监控 Tinc 进程状态
- 自动重启异常退出的 Tinc 进程
- 记录运行日志
- 提供服务管理接口

#### 3.5.3 技术实现
- 使用 QtService 框架实现跨平台服务
- Windows: 注册为 Windows 服务
- Linux: 注册为 systemd 服务

#### 3.5.4 服务管理命令
**Windows:**
```bash
# 安装服务
Daemons.exe -i

# 启动服务
Daemons.exe -s

# 停止服务
Daemons.exe -t

# 卸载服务
Daemons.exe -u
```

**Linux:**
```bash
# 调试模式运行
./Daemons -exec

# 安装为系统服务
./Daemons -i

# 启动服务
./Daemons -s
```

## 4. 配置文件说明

### 4.1 private.txt
存储系统配置信息，位于 `d:/Codes/Java/KenDeJi_RuoYi/tinc_cli_gui/windows/`

**格式示例:**
```
net_name:Hello
server_ip:127.0.0.1:8081
node_ip:192.168.103.34
```

**字段说明:**
- `net_name`: Tinc 网络名称
- `server_ip`: 服务器 IP 地址（包含端口号）
- `node_ip`: 节点 IP 地址

### 4.2 Tinc 配置文件

#### 4.2.1 tinc.conf
Tinc 主配置文件，位于 `Tinc/{网络名称}/`

**示例内容:**
```
Name = dfdfd
Interface = tinc0
Mode = router
ConnectTo = server_master
```

**字段说明:**
- `Name`: 节点名称
- `Interface`: 网络接口名称
- `Mode`: 运行模式（router/switch/hub）
- `ConnectTo`: 连接的服务器节点

#### 4.2.2 tinc-up
网络接口启动脚本

**示例内容:**
```bash
#!/bin/sh
ifconfig $INTERFACE 192.168.103.34 netmask 255.255.255.0
```

## 5. 编译与部署

### 5.1 开发环境要求

#### 5.1.1 Windows 平台
- **操作系统**: Windows 7/10/11
- **Qt 版本**: Qt 5.15.2 或更高

## 6. API 接口说明

### 6.1 登录接口

**请求地址:** `POST /login`

**请求参数:**
```json
{
  "username": "用户名",
  "password": "密码"
}
```

**响应示例:**
```json
{
  "code": 200,
  "msg": "操作成功",
  "token": "sun-token-1234567890"
}
```

### 6.2 下载配置接口

**请求地址:** `POST /downloadConfig`

**请求参数:**
```json
{
  "sid": "节点ID",
  "token": "认证令牌"
}
```

**响应:** ZIP 格式的配置文件压缩包

### 6.3 上传公钥接口

**请求地址:** `POST /XVntQFJCjc.php/coreplugs/Clientinterface/exchangeFile`

**请求参数:**
```json
{
  "sid": "dfdfd",
  "token": "sun-token-1772437758839",
  "netName": "Hello",
  "nodeIp": "127.0.0.1",
  "action": "add",
  "content": "-----BEGIN RSA PUBLIC KEY-----\n...\n-----END RSA PUBLIC KEY-----"
}
```

**响应示例:**
```
Address = 122.152.232.241
Subnet = 192.168.103.1/32

-----BEGIN RSA PUBLIC KEY-----
MIICIjANBgkqhkiG9w0BAQEFAAOCAg8AMIICCgKCAgEAvY4YknoWpgjzKcfe/wEx...
-----END RSA PUBLIC KEY-----
```

## 7. 使用说明

### 7.1 首次使用流程

#### 7.1.1 Windows 平台

1. **启动登录程序**
   ```bash
   cd C:/TincClient/bin
   LoginDialog.exe
   ```

2. **输入登录信息**
   - 用户名: 系统分配的用户名
   - 密码: 对应的密码

3. **开始配置**
   - 点击"开始配置"按钮
   - 等待配置完成（约 30-60 秒）
   - 查看配置日志确认成功

4. **启动 VPN**
   - 配置完成后，VPN 会自动启动
   - 或手动启动: `net start TincDaemon`

#### 7.1.2 Linux 平台

1. **启动登录程序**
   ```bash
   /opt/tincclient/bin/LoginDialog
   ```

2. **输入登录信息**
   - 用户名: 系统分配的用户名
   - 密码: 对应的密码

3. **开始配置**
   - 点击"开始配置"按钮
   - 等待配置完成
   - 查看配置日志确认成功

4. **启动 VPN**
   ```bash
   sudo systemctl start tinc@Hello
   ```

### 7.2 故障排查

#### 7.2.1 查看日志

**客户端日志:**
- 位置: `conf_package.exe` 所在目录
- 文件名: `conf_log.txt`

**Tinc 日志:**
**Windows:**
```bash
type C:\TincClient\Tinc\Hello\log.txt
```

**Linux:**
```bash
sudo cat /var/log/tinc.Hello.log
```

#### 7.2.2 常见问题

**问题 1: 无法连接服务器**
- 检查网络连接
- 确认服务器 IP 和端口正确
- 检查防火墙设置

**问题 2: 密钥生成失败**
- 检查 tincd.exe 是否存在
- 确认有足够的权限创建文件
- 查看日志了解详细错误

**问题 3: 公钥上传失败**
- 检查服务器是否正常运行
- 确认 token 是否有效
- 查看服务器日志

**问题 4: VPN 无法启动**
- 检查 TAP 驱动是否安装（Windows）
- 确认配置文件正确
- 检查端口是否被占用

## 8. 安全注意事项

### 8.1 密钥安全
- 私钥文件（rsa_key.priv）必须严格保密
- 不要将私钥文件发送给任何人
- 定期更换密钥对

### 8.2 网络安全
- 使用 HTTPS 协议进行通信
- 验证服务器证书
- 使用强密码

### 8.3 系统安全
- 定期更新系统和软件
- 及时安装安全补丁
- 限制不必要的网络访问

### 8.4 日志管理
- 定期清理日志文件
- 不要在日志中记录敏感信息
- 保护日志文件的访问权限
